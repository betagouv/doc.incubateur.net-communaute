# Les bases de la documentation

### **C’est quoi une documentation utile ?** <a href="#cest-quoi-une-documentation-utile" id="cest-quoi-une-documentation-utile"></a>

* Une documentation utile doit être conçue de façon à répondre aux besoins de ceux qui la lise.
* Une documentation utile n’a pas besoin d’être dense ou exhaustive.

### **Quelles sont les quatre questions à se poser avant de démarrer ?** <a href="#quelles-sont-les-quatre-questions-a-se-poser-avant-de-demarrer" id="quelles-sont-les-quatre-questions-a-se-poser-avant-de-demarrer"></a>

* Comment documenter ?
* Pour qui documenter ?
* Quoi documenter ?
* Où documenter ?

### **À quoi faisons-nous référence quand on parle de documentation ?** <a href="#a-quoi-faisons-nous-reference-quand-on-parle-de-documentation" id="a-quoi-faisons-nous-reference-quand-on-parle-de-documentation"></a>

* FAQ
* Documentation
* Notice
* Guide d’utilisation
* Wiki
* ...

Ces termes sont souvent utilisés mais d’un produit à l’autre ou d’une équipe à l’autre, leur forme et leur sens sont différents.

### **Quels sont les principaux problèmes rencontrés par une documentation mal pensée ?** <a href="#quels-sont-les-principaux-problemes-rencontres-par-une-documentation-mal-pensee" id="quels-sont-les-principaux-problemes-rencontres-par-une-documentation-mal-pensee"></a>

Pour les lecteurs :

* Elle n’est pas à jour
* Elle n’est pas utile
* Elle n’est pas organisée
* Elle ne répond pas aux attentes
* Elle est confuse
* Elle est trop dense

Pour les auteurs :

* Elle est très chronophage à réaliser
* Elle n’est pas utilisée
* Il est difficile de la tenir à jour
